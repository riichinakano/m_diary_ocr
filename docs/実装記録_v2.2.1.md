## v2.3.0 追加実装（2026-05-10）

### A. Google Sheets API 安定化（`core/csv_store.py`）

#### A-1. キャッシュ導入
- `read_visits()` / `read_sales()` に `@st.cache_data(ttl=60)` を追加。
- ウィジェット操作のたびに API を呼び出していた問題を解消し、60秒に1回の呼び出しに抑制。

#### A-2. リトライロジック
- `_read_gsheet` を最大3回リトライ（間隔1秒）するよう書き換え。
- **重要設計**：最終失敗時は空の DataFrame を返さず `raise` する。
  - `@st.cache_data` は例外を返した呼び出しをキャッシュしない仕様を活用し、API 回復後に自動で再試行できるようにした。
  - 旧実装（`except: return pd.DataFrame()`）は空 DataFrame がキャッシュされ、API が回復しても60秒間は古いキャッシュを返し続けるバグがあった。

#### A-3. 全書き込み操作後のキャッシュクリア
- `upsert_visits` / `upsert_sales` / `delete_visit` / `delete_sale` の末尾に `read_visits.clear()` / `read_sales.clear()` を追加。
- 書き込み直後にページ再読み込みしても古いキャッシュが表示されない。

#### A-4. `delete_by_date()` 関数追加
```python
def delete_by_date(date: str) -> tuple[int, int]:
    """指定日のすべての入館者・物販データを削除する。削除件数を (visits, sales) で返す。"""
```
- 全期間データ（月フィルタなし）に対して削除するため、別月への誤入力も修正可能。

### B. 履歴ページの機能追加（`pages/2_📋_履歴.py`）

#### B-1. 🔄 更新ボタン
- ヘッダー右端に「🔄 更新」ボタンを追加。
- キャッシュを強制クリアして即時再取得（`read_visits.clear()` → `st.rerun()`）。

#### B-2. API 障害時の graceful degradation
- `read_visits()` / `read_sales()` を try/except で囲み、失敗時はエラーメッセージ表示 + `st.stop()`。
- ページが壊れた状態で表示され続けることを防ぐ。

#### B-3. 🗑️ 日付一括削除（Expander）
- 全期間のすべての日付をセレクトボックスで選択し、削除件数をプレビューしてから一括削除。
- `csv_store.delete_by_date()` を呼び出す。入力日を間違えた場合の修正に対応。

#### B-4. 📊 期間指定集計（Expander）
- 開始日・終了日を日付ピッカーで指定し、その期間の集計値を月次集計と同じ指標で表示。
- 月またぎ（例：2026/4/1 〜 2026/5/31）にも対応。全期間データから日付文字列比較で絞り込む。
- デフォルト：当月1日〜今日。開始日 > 終了日の場合は警告表示。

## v2.2.1 実装（2026-05-08）

## 概要
要件定義書 v2.2 に基づくデータ整合性の修正、UIの改善、および Streamlit Community Cloud での安定運用に向けたデータ保存先の Google Sheets 移行を実施しました。

---

## 1. UI・データ整合性の改善（v2.2.1 修正指示への対応）

### 1-1. 設定ページのバグ修正 (`pages/3_⚙️_設定.py`)
- マスターデータ（展覧会、物販、割引、入館者分類）の編集・削除時の不具合を解消しました。
- 削除処理を `delete_idx` パターンに統一し、行を削除した際に「編集中の他の行のデータ」が失われないよう、データの整合性を担保しました。

### 1-2. 入力ページのバリデーション (`pages/1_📝_入力.py`)
- 入力矛盾を防ぐため、カテゴリ名に「（割引）」または「（特別優待）」が含まれているにもかかわらず、割引種別が「（なし）」のまま保存しようとした場合、エラーを表示して追加をブロックする処理を実装しました。

### 1-3. 履歴ページの集計動的化 (`pages/2_📋_履歴.py`)
- 物販集計における品名（`P：絵葉書` など）のハードコーディングを解消しました。
- Pandas の `groupby` を用いて、マスターデータの変更や新規商品の追加に自動で追従する動的集計へ変更しました。

### 1-4. スマートフォン向けUIの最適化 (`pages/2_📋_履歴.py`)
- 入力履歴の表示を `st.columns` による縦並びから、横スクロール可能な `st.dataframe`（テーブル表示）に変更し、スマホでの視認性を大幅に向上させました。
- これに伴い、行単位の削除操作を Expander（折りたたみメニュー）内のセレクトボックスから選択して実行する形式に変更しました。

---

## 2. データ保存先のクラウド移行（Google Sheets 連携）

### 2-1. 保存先の変更 (`core/csv_store.py`)
- Streamlit Community Cloud などのエフェメラル（一時的）な環境における「再起動による CSV データの消失」を防ぐため、データの保存先をローカルの CSV ファイルから **Google スプレッドシート** へ完全移行しました。
- `gspread` および `google-auth` を導入し、Streamlit の `secrets.toml` に設定したサービスアカウントを通じてシートを読み書きする方式を採用しました。

### 2-2. 関数インターフェースの整理
- ローカルディレクトリへの保存が不要になったため、`core/csv_store.py` の各関数の引数から `data_dir` を削除し、コードを整理しました（`pages/1_📝_入力.py`, `pages/2_📋_履歴.py` もそれに合わせて修正）。

### 2-3. エラーハンドリングの強化 (`core/csv_store.py`)
- 権限不足や通信エラーによるサイレントエラーを防ぐため、スプレッドシートへの書き込み (`_write_gsheet`) 失敗時に `st.error` で画面に詳細なエラー内容を表示するようにしました。
