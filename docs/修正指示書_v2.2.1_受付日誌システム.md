## 2. 🔴 修正1：`pages/3_⚙️_設定.py` の削除と編集の整合性バグ

### 2-1. 問題の説明

現状コード（展覧会リスト部分の例）：

```python
for i, exh in enumerate(exhibitions):
    c1, c2 = st.columns([6, 1])
    val = c1.text_input(..., value=exh, key=f"exh_val_{i}")
    updated_exh.append(val)
    if c2.button("削除", key=f"del_exh_{i}"):
        exhibitions.pop(i)              # ← 編集中のupdated_exhを捨ててしまう
        master["exhibitions"] = exhibitions
        _save(master)
        st.rerun()
```

**問題点**：
1. ユーザーが行Aの値を編集してから行Bを削除した場合、行Aの編集内容は保存されずに失われる
2. ループのインデックス `i` ベースでpopしているため、Streamlitの再描画タイミング次第で意図しない行が削除される可能性がある

### 2-2. 修正方針

**統一ルール**：
1. 削除ボタン押下時は、その時点で `updated_*` リスト（編集中の最新状態）から該当インデックスを除外して保存する
2. 削除ボタンの key は安定識別子（コードや値ベース）を使う
3. 編集中の値を必ず保存対象に含める

### 2-3. 具体的な修正コード

#### ① 展覧会リスト

```python
# ── 展覧会リスト ──
with st.expander("🎨 展覧会リスト", expanded=True):
    exhibitions: list[str] = list(master.get("exhibitions", []))
    updated_exh: list[str] = []
    delete_idx: int | None = None

    for i, exh in enumerate(exhibitions):
        c1, c2 = st.columns([6, 1])
        val = c1.text_input(
            f"展覧会 {i + 1}", value=exh,
            label_visibility="collapsed", key=f"exh_val_{i}",
        )
        updated_exh.append(val)
        # 削除ボタンはフラグだけ立てて、ループ後にまとめて処理
        if c2.button("削除", key=f"del_exh_{i}"):
            delete_idx = i

    # 削除処理（編集中の値を保持したまま）
    if delete_idx is not None:
        new_list = [e.strip() for j, e in enumerate(updated_exh)
                    if j != delete_idx and e.strip()]
        master["exhibitions"] = new_list
        _save(master)
        st.rerun()

    st.markdown("---")
    c_new, c_add = st.columns([6, 1])
    new_exh = c_new.text_input(
        "新規展覧会名", placeholder="例: 9_夏季展_前期",
        label_visibility="collapsed", key="new_exh_input",
    )
    if c_add.button("追加", key="btn_add_exh"):
        trimmed = new_exh.strip()
        if trimmed and trimmed not in updated_exh:
            # 追加時も編集中の値を保持
            new_list = [e.strip() for e in updated_exh if e.strip()]
            new_list.append(trimmed)
            master["exhibitions"] = new_list
            _save(master)
            st.rerun()
        elif trimmed in updated_exh:
            st.warning("既に登録されています")

    if st.button("展覧会リストを保存", key="save_exh_btn"):
        master["exhibitions"] = [e.strip() for e in updated_exh if e.strip()]
        _save(master)
```

### 2-4. 変更パターンの要約（4セクション共通）

すべてのリスト編集セクションで以下のパターンに統一：

```
1. ループ内：text_input 等の値を updated_* リストに append
2. ループ内：削除ボタンは delete_idx にインデックスをセットするだけ
3. ループ後：delete_idx が None でなければ、updated_* から該当行を除外して保存・rerun
4. 「追加」ボタン：新規値を updated_* リストに加えて保存・rerun（編集中の値は updated_* から取得）
5. 「全体保存」ボタン：updated_* をそのまま保存
```

### 2-5. テスト観点

修正後、以下の動作を確認すること：

- [ ] 行Aの値を編集中に行Bを削除 → 行Aの編集内容が保持される
- [ ] 入館者分類で「一般」の料金を800円に変更してから別の行を削除 → 800円が反映される
- [ ] 連続して複数回削除を行っても意図した行のみ消える
- [ ] 編集中に「追加」ボタンを押しても編集内容が保持される

## 3. 🟡 修正2：`pages/1_📝_入力.py` の category/discount 整合性

### 3-1. 問題の説明

現状、ユーザーが「一般（割引）」を選んでいるのに割引種別で「（なし）」を選択することができてしまう：

```csv
2026-05-01,…,A:一般,一般（割引）,,2     ← 「割引」と書いてあるのに割引なし
```

これは矛盾入力。

### 3-2. 修正方針

`category` 文字列に「（割引）」または「（特別優待）」が含まれている場合は、`discount` の選択を必須にする（「（なし）」を選ぶとエラー）。

### 3-3. 具体的な修正コード

入館者の「追加」ボタンクリック処理を以下のように変更：

```python
with col_add:
    if st.button("追加", key="btn_add_v", use_container_width=True):
        vc   = visitor_cats[v_cat_idx]
        disc = "" if v_disc_idx == 0 else discounts_all[v_disc_idx - 1]

        # category と discount の整合性チェック
        needs_discount = ("（割引）" in vc["category"]
                          or "（特別優待）" in vc["category"])
        if needs_discount and not disc:
            st.error(
                f"「{vc['category']}」を選択した場合は、割引種別を選んでください"
            )
        else:
            st.session_state.visitors.append({
                "code":     vc["code"],
                "category": vc["category"],
                "discount": disc,
                "count":    int(v_count),
            })
            st.rerun()
```

### 3-4. （オプション）逆方向のチェックも追加する場合

「割引なし category」（例：「一般」「シニア」「招待券」）に対して discount が選ばれた場合も警告すべき。シンプルに以下を追加：

            st.warning(
                f"「{vc['category']}」は割引なしの分類ですが、"

エラーにせず警告に留める理由：
- 「招待券」+「団体」のような正当な組み合わせの可能性も残る
- 運用しながら判断する余地を持たせる

### 3-5. テスト観点

- [ ] 「一般」（割引なし）+「（なし）」→ 追加できる
- [ ] 「一般（割引）」+「HP」→ 追加できる

- [ ] 「一般（特別優待）」+「（なし）」→ エラーで追加できない
