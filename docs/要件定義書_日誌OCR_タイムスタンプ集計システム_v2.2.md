## 0. v1.0からの変更点（最終確定版）

| 項目 | v1.0 | v2.2 |
|------|------|------|
| **入館者カウント** | 手書き日誌をOCRで読み取り | iOS Streamlitアプリで人手入力 |
| **物販記録** | 手書き日誌をOCRで読み取り | iOS Streamlitアプリで人手入力 |
| **連絡事項（AM/PM）** | OCRで構造化抽出 | OCRで全文Markdown化（精度不問・参照用） |
| **GUI** | Tkinterデスクトップアプリ | **Streamlit（入力）+ Tkinter（OCR）の二本立て** |
| **集計・分析** | `json_to_excel.py`で自動追記 | **Claude for Excelに委譲**（CSV手渡し） |
| **中間データ** | JSON（構造化） | CSV（visits.csv / sales.csv） |
| **アクセス方法** | PC直接操作 | 同一LAN上のiPhoneブラウザ + 受付PC |

### 設計思想：適材適所の分離
v2.2では**「ツールを統合せず、得意分野で分担する」**方針を採用：

| 担当ツール | 役割 | 実行場所・頻度 |
|----------|------|---------------|
| **Streamlit** | データ入力・履歴閲覧 | iPhone・日次 |
| **Claude for Excel** | 集計・グラフ・既存Excel更新 | PC・月次 |
| **Tkinter版OCR** | 日誌PNGの全文Markdown化 | PC・月次 |

これにより、各ツールが小さく独立し、メンテナンス性と運用柔軟性を確保する。

## 1. プロジェクト概要

美術館の受付日誌の運用を以下の3系統に分割する：

1. **入力系（Streamlit）**：入館者数・物販販売数を、館内Wi-Fi経由でiPhoneから入力 → CSV
2. **集計系（Claude for Excel）**：CSVを既存Excelに統合し、集計・分析を実施
3. **メモ系（Tkinter OCR）**：手書き日誌PNGを全文Markdown化して保管（精度不問）

各系統は疎結合で、CSV/Markdownファイルを介してのみ連携する。

---

## 2. 業務フロー

```
[Step 1] 入館者・物販の入力（毎日 or 数日まとめて）
  iPhone Safari → http://<受付PCのIP>:8501
  → 日付選択 → 分類×割引×人数を入力 → 保存
  → visits.csv / sales.csv に1行追記

[Step 2] 日誌画像の撮影・保管（従来どおり）
  Scanner Pro (iOS) で受付日誌を撮影
  → OneDrive 月別フォルダにPNG保存

[Step 3] 月次OCR（参照用テキスト化、月1回）
  受付PCで Tkinter版 diary_ocr.py 実行
  → YYYYMMnote.md（全文Markdown、参照用）

[Step 4] 月次集計（月1回）
  受付PCで Claude for Excel を起動
  → visits.csv / sales.csv を渡す
  → 既存 R7_2025年度入館者一覧.xlsx に追記・集計依頼

[Step 5] 目視確認・レポート作成
  Excel上で集計結果を確認
```

## 5. データ構造

### 5-1. visits.csv（入館者）

**設計方針**：1日×1分類×1割引パターンで1行に集約。Claude for Excelで読み込みやすいシンプル構造。

```csv
date,exhibition,code,category,discount,count,created_at,updated_at
2026-05-01,8_春季展_前期,A：一般,一般,,5,2026-05-01T18:30:00,2026-05-01T18:30:00
2026-05-01,8_春季展_前期,A：一般,一般（割引）,HP,2,2026-05-01T18:30:00,2026-05-01T18:30:00
2026-05-01,8_春季展_前期,B：シニア,シニア（割引）,HP,3,2026-05-01T18:30:00,2026-05-01T18:30:00
2026-05-01,8_春季展_前期,F：招待券,招待券,,1,2026-05-01T18:30:00,2026-05-01T18:30:00
```

| 列名 | 型 | 説明 |
|------|-----|------|
| `date` | YYYY-MM-DD | 開館日 |
| `exhibition` | str | 展覧会名 |
| `code` | str | 入館者コード（A：一般など） |
| `category` | str | 入館者分類（既存マスター準拠） |
| `discount` | str | 割引種別（HP/JAF等、なければ空） |
| `count` | int | 人数 |
| `created_at` | ISO8601 | 初回作成日時 |
| `updated_at` | ISO8601 | 最終更新日時 |

**一意キー**：`(date, code, category, discount)`

### 5-2. sales.csv（物販）

```csv
date,exhibition,code,name,count,created_at,updated_at
2026-05-01,8_春季展_前期,P：絵葉書,絵葉書（＠100円）,3,2026-05-01T18:30:00,2026-05-01T18:30:00
2026-05-01,8_春季展_前期,Q：図録,作品選,1,2026-05-01T18:30:00,2026-05-01T18:30:00
```

| 列名 | 型 | 説明 |
|------|-----|------|
| `date` | YYYY-MM-DD | 開館日 |
| `exhibition` | str | 展覧会名 |
| `code` | str | 図版コード |
| `name` | str | 商品名 |
| `count` | int | 販売数 |
| `created_at` | ISO8601 | 初回作成日時 |
| `updated_at` | ISO8601 | 最終更新日時 |

**一意キー**：`(date, code, name)`

### 5-3. メタ情報CSV（オプション）

スタッフ・天気・連絡事項メモなどを記録する場合は、別途 `daily_meta.csv` を分ける案も検討：


## 6. OCR機能（Tkinter継続使用）

### 6-1. 既存`diary_ocr.py`からの変更点

**変更1：プロンプトの大幅簡素化**

旧プロンプト（構造化表形式）から、全文素直な書き起こしに変更：

```python
PROMPT_SIMPLE = """\
この画像は美術館受付日誌の1ページです。
書かれている内容を可能な範囲でMarkdown形式で書き起こしてください。

【ルール】
- 日付があれば ## 見出しに
- 表は表として、メモは段落として
- 読めない箇所は [?] と記載
- 罫線・装飾は無視して、内容のみ抽出
- スタッフ名・天気・連絡事項などはそのまま
- 入館者数の正確な転記は不要（メモ程度でよい）
"""
```

**変更2：構造化JSON出力の廃止**
- `md_to_json.py` は **削除**
- 出力は `YYYYMMnote.md` のみ（参照用テキスト）

**変更3：Tkinter UIのSimplification**
- 「OCR実行」ボタンのみ残し、JSON変換ボタンは削除
- 既存`app.py`から不要な機能を削除し、`diary_ocr_gui.py`にリネーム

## 7. Claude for Excel 連携

### 7-1. 月次集計フロー

```
1. 受付PCで Claude for Excel を起動
2. visits.csv / sales.csv をアップロード
3. 既存 R7_2025年度入館者一覧.xlsx を開く
4. プロンプト例：
   「visits.csvの2026年5月分を、R7_入館者シートの末尾に追記してください。
    - 既存行と日付が重複する場合はスキップ
    - 入館料単価はリストシートからVLOOKUPで参照
    - 入館料合計は単価×人数で計算」
5. Claude for Excelが自動で追記・数式設定
6. 結果を目視確認 → 保存
```

### 7-2. Claude for Excel に渡す指示テンプレート（参考）

```markdown
# visits.csv の Excel追記依頼

## 入力
- visits.csv（添付）
- 既存ファイル: R7_2025年度入館者一覧.xlsx

## 作業
R7_入館者シートの末尾に visits.csv の各行を追記してください。

### 列マッピング
- A列: date
- B列: code
- C列: category
- D列: discount（空文字ならセル空白）
- E列: count
- F列: =IF(C{r}="","",IFERROR(VLOOKUP(C{r},リスト!$B$3:$C$18,2,FALSE),""))
- G列: =IF(OR(E{r}="",F{r}=""),"",E{r}*F{r})
- H列: exhibition

### 重複チェック
A列の同じ日付の行が既存にある場合はスキップしてください。

### 出力
更新後のExcelファイルを保存してください。
```

### 7-3. 集計依頼の例

入力後、追加で集計を依頼可能：

```
「2026年5月の入館者を、分類別・割引別に集計して棒グラフにしてください」
「展覧会別の物販売上TOP10を表にしてください」
「曜日別の平均入館者数を計算してください」
```

→ Streamlit側に集計画面を作る必要がない。

## 13. 設計上の判断記録

| 判断項目 | 採用 | 棄却した代替案 | 理由 |
|---------|------|-------------|------|
| 入力UI | Streamlit | Tkinter継続 | iOS対応・Webブラウザで動く |
| ホスティング | ローカルPC + LAN | Streamlit Cloud | Cloudはファイル永続化されずCSV消失 |
| データ形式 | CSV | SQLite / JSON / Sheets | シンプル・Claude for Excel直接読込可 |
| 入力粒度 | 1日1分類で集約 | タイムスタンプ付きログ | 後追い入力のため時刻情報の意味薄 |
| 集計手段 | **Claude for Excel** | Streamlit内集計画面 / openpyxl自動追記 | 既存Excel資産活用・自前実装ゼロ・柔軟性最大 |
| OCR役割 | 全文メモのみ | 構造化抽出継続 | 手書き表の読み取り精度が業務基準未達 |
| OCR実装 | **Tkinter継続** | Streamlit統合 / Claude Skill | 月1回実行・既存資産流用・実装コスト最小 |
| 機能配置 | 入力=Streamlit、集計=Excel、OCR=Tkinter | 全機能Streamlit統合 | 適材適所で各ツール小さく独立 |

## 14. v2.2の設計哲学

### 「統合」より「疎結合」
v1.0はTkinterで全機能を統合していたが、v2.2では機能ごとに最適なツールを選択：

- **入力に強い**: Streamlit（Web UI、iOS対応）
- **集計に強い**: Claude for Excel（既存Excel活用、自然言語で柔軟操作）
- **OCRバッチに強い**: Tkinter（軽量、ローカル完結、既存資産）

これらを **CSV/Markdownファイル** という単純なインターフェースで連携させる。

### 「自前実装」より「既存ツール活用」
集計画面・グラフ・ピボットを自前実装するコストを排除し、Claude for Excel に委譲。Streamlitはひたすら入力UIに専念する。

### 「精度向上」より「人手分担」
OCRの精度を上げる努力をやめ、定量データは人手入力、定性メモはOCR全文化、と役割を分けた。これにより両方とも要求水準を100%満たす。
